# Visão geral do sistema
Sistema para cadastro de sistemas e suas dependências.

## 1. Objetivo do projeto
Desenvolver um sistema multiagente para apoiar a criação, análise e implementação de um software. Este software terá como tema a estrutura de sistemas de uma gerência de uma determinada empresa, providênciando rastreabilidade e análise técnica de mudanças.

## 2. Problema que o sistema resolve
O sistema visa apoiar as mudanças em ambiente de produção com maior segurança, clareza e rastreabilidade. O sistema deve mapear os sistemas de uma gerência e suas interrelações, chamadas diretas ou por API. O sistema irá automatizar um processo realizado manualmente por chat.

## 3. Atores envolvidos
O sistema será utilizado principalmente pelo gestor, que deve fazer a análise da mudança em ambiente produtivo proposta. Os demais desenvolvedores da gerência, utilizarão o sistema para requisitar as mudanças em ambiente de produção.

## 4. Escopo inicial
A proposta do sistema é simples e não muito grande, o escopo inicial conta com a completude do sistema base, podendo haver feedbacks futuros dos usuários que irão incrementar a implementação.

## 5. Restrições técnicas
O backend do sistema deverá ser desenvolvido em PHP 8.2, será realizada uma análise de qual tipo de banco de dados utilizar (relacional, documentos, grafos) e qual SGBD para acompanhar esta decisão. O frontend será desenvolvido mobile com a tecnologia React Native e as bibliotecas necessárias.

## 6. Pedido para a IA
Atue como arquiteto de software. Analise o cenário e proponha a melhor estrutura inicial, considerando arquitetura, módulos, dependências e riscos.
