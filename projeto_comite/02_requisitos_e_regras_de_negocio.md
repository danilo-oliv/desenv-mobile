# Requisitos e regras de negócio

## 1. Requisitos funcionais
[RF-01] Os usuários em geral devem poder manter sistemas. Um sistema contém nome, código (alfanumérico), responsável técnico (um ou mais), dependentes (sistemas que possuem alguma dependência do sistema em questão), dependências (sistemas que são necessários para o funcionamento do sistema em questão), data da última atualização, informações da última publicação (data, data de autorização do comitê, tag publicada, responsável da publicação)

[RF-02] Os usuários desenvolvedores devem poder criar uma solicitação de aprovação de mudança informando: sistema, tag, data de publicação, responsável, descrição.

[RF-03] Os usuários membros do comitê devem poder visualizar todas as solicitações de aprovação de mudança.

[RF-04] Os usuários membros do comitê devem poder aprovar ou reprovar as solicitações de aprovação de mudança e adicionar uma justificativa.
